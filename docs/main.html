Getting Started

The first thing you need to do to get started with smash scripting, is check your system for compatibility with smash and the smash build system. The following table lists the build system dependencies of the smash project. We will cover the smash-software-run-time-requiem after satisifcation of the smash-software-build-time-requiems are known.

The first field in the table is a reference to an external project by it's official project name or official package name. Official project name is preffered, followed by a 'generified' package-name within parenthesis. The Dev. Version field, refers to the current version of the program run on the machine of the smash-project-maintainer(s).

Finally, the notes field provides any extra information that may be beneficial knowledge about the external project.

Smash Build System Requirements

Package Dev. Version Notes
GNU Compiler Collection (GCC) 8.2.1 r20180831 (ArchLinux)
GNU Make 4.2.1
GNU Automatic Configuration Tools (autoconf) 2.13 autoconf version 2.13 is required by the Mozilla js-1.8.5 buid system.
GNU Bourne Again Shell (BASH) 4.4

This is a general, rather than complete listing of dependencies. A basic build environment is required for compilation of all C/C++ programs and libraries.
